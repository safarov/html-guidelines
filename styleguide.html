Стандарты верстки компании JetStyle

Рекомендации по оформлению кода

Данный гайд задает общие правила и синтаксис при создании HTML и CSS, использование данного синтаксиса поможет избежать ошибок при компиляции кода препроцессорами перед выгрузкой в продакшн и поможет другим разработчикам проще ориентироваться в вашем коде.

Общие правила форматирования

Отступы

Для отступов не рекомендуется использовать табы или смесь табов и пробелов. В разных редакторах для табов может быть настроена разная длина, что может сильно затруднить чтение кода на другом компьютере.

Стандартный отступ — 4 пробела. О занимаемом месте волноваться не надо, при выгрузке в продакшен мы все равно будем очищать код от разного мусора.

Регистр

Пишем названия тегов, аттрибутов, селекторов и значений свойств (кроме строковых) с маленькой буквы. Больше единообразия, меньше путаницы и если вдруг мы когда-нибудь заходим сделать код валидным xhtml, то будет меньше возни.

Комментарии

По мере необходимости и в рамках разумного пишем комментарии, если вам кажется что назначение какого-то куска кода или причины выбора того или иного решения могут быть неочевидными, лучше их прокомментировать. Комментировать все не нужно.

Пометки в комментариях

Места для будущих доработок помечаются ключевым словом TODO в комментарии. Описание того что нужно исправить пишется после двоеточия. Пример:

<!-- TODO: Вынести в отдельный блок -->

Опционально можно писать в скобках после TODO имя того кто пинать на предмет исправления. Пример:

<!-- TODO(Иванов): Вынести в отдельный блок -->

Стилевые правила HTML

Отделение представления от содержания

Все стили только в CSS. Если JavaScript изменяет вид страницы, желательно делать это при помощи смены CSS-классов, а не путем правки свойства style элемента. Смена класса обрабатывается браузером гораздо быстрее и отменяется гораздо проще.

Использование entity

У нас тут UTF-8, поэтому большинство символом можно со спокойной совестью писать без изпользования entity. Исключения символы <, >, & и символы котроые без обозначения как entity не будут видны в коде (например &nbsp;).

Необязательные теги

HTML5 повзоляет опускать некоторые теги как необязательные. Например <html>, <head>, <body>,</p> в целях экономии места и ускорения загрузки. Мы постараемся автоматизировать подобную обработку перед отправкой страниц в браузер, но при разработки опускать их не рекомендуется, так как это ухудшает читаемость документов.

Аттрибут type в тегах <link> и <script>

При подключении css- и js-файлов не надо писать type="test/css" и type="text/jsvascript", так как эти значения используются браузерами по-умолчанию. type необходимо использовать при использовании препроцессоров.

Выделение аттрибутов

Для аттрибутов всегда используем двойные кавычки.

Расставление переносов

Для элементов уровня блока, списков и таблиц теги нужно писать с новой строки, для инлайновых элементов, можно оставлять на месте.

Правила написания CSS

Пути до ресурсов и сложные значения с пробелами

Пути до ресурсов и сложные значения (например имена шрифтов с пробелмами) пишем в двойных кавычках.

Хаки

По возможности не используем хаки, вместо них пробуем определять IE при помощи conditional комментариев.

Форматирование CSS

Порядок и расположение свойств

Свойства необходимо начинать с новых строк, это облегчит их поиск другим разработчиками.

Закрывающую фигурную скобку необходимо ставить на том же уровне, что и определение селектора, переносить длинные css-свойства посередине нельзя. Это необходимо для корректной работы css-препроцессоров. Пример:

.b-block {
    display: block;
    position: relative;
    left: 0;
}

Порядок написания отдельных селекторов можно выбирать по своему вкусу. Для более удобной работы с файлами других разработчиков рекомендуется создать свой конфиг для csscomb и использовать него по необходимости.

CSS-префиксы

Для упрощения написания и поддержки кода, рекомендуется использовать для записи свойств CSS3 синтаксис Stylus’а.

Точка с запятой

Точку с запятой необходимо использовать после объявления каждого правила чтобы избежать проблем при изменении порядка сортировки.

Названия свойств

После названия свойств и двоеточия следует ставить пробел, для облегчения чтения. Лишние символы вырежет компрессор.

Оформление составных селекторов

При использовании составных селекторов (состоящих из нескольких элементов, разнесенных запятыми), необходимо начинать каждый селектор с новой строки, для облегчения чтения. Пример:

h1,
h2,
h3 {
    display: block;
}

Разделение свойств

Отделяйте свойства друг от друга пустой строкой, для более удобного чтения и поиска.
